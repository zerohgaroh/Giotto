# ЧАСТЬ I — ДИЗАЙН-СИСТЕМА

## 1. Источник айдентики

Дизайн платформы строго следует фирменному стилю ресторана Giotto. Логотип — круговая эмблема на глубоком тёмно-синем фоне с белым леттерингом, монограммой «G» и дуговыми акцентами. Стиль — итальянский классицизм с динамичным характером: уверенный, но элегантный.

Все интерфейсы (гость, официант, менеджер, владелец) должны ощущаться как единая система, рождённая из этого логотипа.

## 7. Анимации и переходы

Сдержанные, итальянский темперамент — без суеты:

```css
/* Стандартный переход */
transition: all 0.2s ease;

/* Появление карточки заказа в панели менеджера */
@keyframes slideIn {
  from { opacity: 0; transform: translateY(-8px); }
  to   { opacity: 1; transform: translateY(0); }
}
animation: slideIn 0.25s ease forwards;

/* Статус заказа (смена иконки) */
transition: color 0.3s ease, background-color 0.3s ease;

/* Bottom sheet (корзина) */
transition: transform 0.35s cubic-bezier(0.34, 1.56, 0.64, 1);
```

Не использовать: резкие bounce-анимации, длительные transitions > 0.4s, мигающие элементы.

# ЧАСТЬ II — ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

## Контекст проекта

Ресторан Giotto — итальянский ресторан среднего и высокого ценового сегмента. Платформа строится на NFC-карточках: гость прикладывает телефон к столику — браузер открывает персонализированное веб-приложение. Никакой установки, никакой авторизации для гостя.

---

## Принцип работы NFC

Каждый стол оснащён NFC-карточкой (NTAG213 или NTAG215) с NDEF-записью:

```
https://giotto.restaurant/table/{table_id}
```

Все последующие действия гостя — заказ, вызов официанта, разделение счёта — автоматически привязаны к `table_id` из URL. HTTPS обязателен: NFC Web API работает только по защищённому протоколу.

Карточки после прошивки переводятся в режим read-only (флаг защиты от перезаписи).

---

## Роли и права доступа

| Роль | Авторизация | Область видимости |
|---|---|---|
| Гость | Нет (URL = идентификатор) | Только свой столик |
| Официант | Логин + пароль (PWA) | Только свои столики (зонирование) |
| Менеджер | Логин + пароль | Все столики, операционная аналитика |
| Владелец | Отдельный аккаунт | Всё + финансовая аналитика, удалённый доступ |

---

## Интерфейс гостя

### Меню
- Цифровое меню с категориями (Antipasti, Insalate, Zuppe, Primi, Secondi, Dolci, Bevande)
- Фото блюда, название на итальянском + перевод, описание, состав, вес, КБЖУ
- Фильтры: вегетарианское, без глютена, острое, рекомендации шефа
- Badge «Giotto рекомендует» — выделяет фирменные блюда
- Стоп-лист: недоступные блюда скрыты или помечены серым с зачёркиванием

### Корзина и заказ
- Добавление блюд с выбором количества
- Комментарий к каждой позиции — прожарка, без лука, острота, аллергия
- Редактирование корзины до отправки
- Отправка заказа одной кнопкой

### Статус заказа (реальное время)
```
Принят → Готовится → Готов → Подаётся
```
Прогресс-бар или шаги — в стиле Giotto (navy + gold). Обновление через WebSocket без перезагрузки страницы.

### Вызов официанта
- Кнопка всегда видна (gold accent, иконка колокольчика)
- Опциональная причина: принести счёт / вопрос / другое
- Подтверждение на экране после нажатия

### Разделение счёта
- Гость инициирует разделение
- Каждый участник отмечает свои блюда из общего заказа
- Система показывает индивидуальную сумму каждого
- Итог по каждому участнику формируется как отдельный мини-счёт

---

## Интерфейс официанта

### Зонирование столов
- Каждый официант видит только свои столики (настраивается менеджером)
- Карточки столов: статус, активные заказы, время с момента посадки

### Уведомления о вызовах
- Push-уведомление (PWA) при нажатии гостем
- Содержит: номер стола, причина, время
- Кнопка «Принято» — закрывает вызов, фиксирует время реакции

### Управление заказами
- Входящий заказ: состав, комментарии гостя, время
- Действия: Принять / Отклонить (с причиной)
- Обновление статуса по этапам: Готовится → Готов → Подаётся

---

## Интерфейс менеджера

### Карта зала
- Визуальная схема с реальным статусом каждого стола
- Статусы: свободен / занят / ожидает заказа / заказ принят / ждёт счёта / вызов официанта
- Клик на стол — детали: гость, заказ, время посадки, назначенный официант

### Управление персоналом
- Добавление/удаление официантов, назначение зон
- Мониторинг смены: столики, время реакции, количество обслуженных гостей

### Управление меню
- Редактор: добавить / изменить / скрыть блюдо
- Стоп-лист: одним переключателем блюдо исчезает из меню гостей
- Загрузка фото, редактирование цен, описаний, состава, КБЖУ

### Операционная аналитика
- Время жизни стола (посадка → счёт)
- Время до первого заказа
- Время реакции официантов на вызов
- Скорость кухни (принят → готово)
- Загрузка столиков, оборачиваемость, часы пик
- Причины и частота вызовов официанта

---

## Интерфейс владельца

Отдельный аккаунт, доступ с любого устройства вне ресторана.

### Финансовая аналитика
- Выручка за день / неделю / месяц / произвольный период
- Средний чек по времени суток и дням недели
- Сравнение периодов
- Топ блюд по выручке и по количеству заказов
- Блюда с высоким процентом удалений из корзины
- Выручка в разрезе официантов

### Операционные отчёты (сводно)
- Средняя оборачиваемость столов по сменам
- Рейтинг официантов по времени реакции
- Экспорт отчётов в PDF / Excel

## Пользовательские сценарии

**Гость:**
